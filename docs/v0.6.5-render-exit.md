# Render-exit arc — extract `render` from the layer SCC (5 → 4)

Status: SPEC. Companion to `docs/v0.6.5-tea-reaudit.md` (F3 layering) and
`docs/reducer-cleanup-relocation.md` (the prior `app`-extraction arc, 6 → 5).

## The question this answers

The residual layer SCC after `app` was extracted is
`{dispatch, feature, overlay, panel, render}` (5 layers). The re-audit blessed
it as needing "a deeper Cmd/port inversion the findings don't justify". A
file-level investigation refines that verdict:

> **`render` is not part of the cycle's engine — it is a passenger.** The
> `render` directory conflates THREE tiers under one name: a leaf (drawing
> primitives), a top (the frame compositor), and a zero-dep seam (the render
> queue). The cycle is held by a separate, deeper `{dispatch, feature, overlay,
> panel}` domain tangle; `render` is dragged in only by (a) leaf primitives
> masquerading as a render peer and (b) `dispatch` calling `render()` directly.

So there are two independent wins, and this arc scopes only the first:

1. **Render-exit (this arc, 5 → 4):** stop `render` being conflated with its
   leaf, and stop `dispatch`/`overlay` reaching up into the compositor. `render`
   becomes a clean top-level compositor that nothing below it imports. Cheap,
   mechanical, behavior-preserving.
2. **Domain detangle (NOT this arc):** dissolve `{dispatch, feature, overlay,
   panel}`. `dispatch↔panel` alone is 43-up / 8-back edges. That is the real
   Cmd/port inversion the docs deferred; out of scope here.

Verified by simulation against `js/scripts/dep-walker.js`'s own parser: after the
moves + four inversions, `render`'s only in-edges are from `app` (top, legal),
and the residual SCC is `{dispatch, feature, overlay, panel}`.

## The unifying mechanism — injected provider seams

Every reach-up is removed the same way the codebase **already** removes the
terminal↔layout cycle: a zero-dependency module in a low layer holds a function
slot that a higher layer registers at boot (`render-queue.setRenderers`,
`pty-lifecycle.setRenderHook`). No behavior changes — only the static import
direction. This arc applies that one blessed pattern uniformly. There are no new
patterns to learn.

## File moves

Extract the leaf tier out of `render/` into `leaves/` (dep-walker layers by
first path segment, so the physical move is what changes the graph). Use
`git mv` to preserve history.

| File | From | To | Tier |
|------|------|----|----|
| `panel.js` (`renderPanel`/`renderOverlay`/`overlayBox`/`truncate`/`viewportDims`) | `render/` | `leaves/draw.js` | leaf primitives (rename: "panel" was a misnomer) |
| `themes.js` | `render/` | `leaves/themes.js` | leaf (palette data) |
| `painter.js` (`composeRows`/`paintFrame`) | `render/` | `leaves/painter.js` | leaf (already pure) |
| `scrollbar.js` | `render/` | `leaves/scrollbar.js` | leaf (already pure, zero-dep) |
| `render-queue.js` | `render/` | `leaves/render-queue.js` | the seam (already zero-dep) |

`decor.js` **splits** (it is two things):
- pure chrome-glyph markup builders (`_collapseGlyphMarkup`, `_closeGlyphMarkup`,
  `_tabTriggerMarkup`) → into `leaves/draw.js` (consumed by `renderPanel`, pure).
- slice-reading hit-test geometry (`chromeFor`, `hitTestCollapseButton`,
  `hitTestCloseButton`, `_placedWidgetTargets`) → `panel/chrome-hittest.js`
  (reads the layout slice via `panel/api`; consumed only by `dispatch/input.js`,
  so `dispatch→panel` — legal and abundant).

**Stays in `render/` = the true compositor (TOP):** `paint.js`, `footer.js`.
These legitimately read down into `panel`/`overlay` (a compositor driving its
components) and are imported only by `app` (top) — both legal in a DAG.

## The four inversions

### ① `leaves/draw.viewportDims` — dims via an injected provider
`viewportDims()` reads `getInstanceSlice('layout').dims` (with an io/term
fallback). A leaf can't read `panel`. Add a provider slot:

```js
// leaves/draw.js
let _dimsProvider = null;
function setDimsProvider(fn) { _dimsProvider = fn; }
function viewportDims() {
  const d = _dimsProvider && _dimsProvider();
  return (d && d.cols > 0 && d.rows > 0)
    ? { cols: d.cols, rows: d.rows }
    : { cols: cols(), rows: rows() };   // io/term fallback (leaf→io, legal)
}
```

`panel/api.js` wires it at module load:
`setDimsProvider(() => getInstanceSlice('layout')?.dims)` — the model-read stays
in `panel` (frame purity preserved: the rendered frame still derives from the
model, never from raw hardware).

### ② `decor` slice-read — resolved by the split (no seam needed)
`_placedWidgetTargets` reads the layout slice. It moves to
`panel/chrome-hittest.js`, which reads `getInstanceSlice` directly (intra-panel).
The pure glyph builders that `renderPanel` needs carry no slice read.

### ③ `overlay → paint.invalidateRows` — through the seam
`overlay/{cmdline,pane-menu}.js` poke the compositor's frame buffer. Route it
through the (now-leaf) render queue: `paint` registers `invalidateRows` into the
seam at boot; overlays import it from `leaves/render-queue` (`overlay→leaves`,
legal).

### ④ `dispatch → paint.render` / `forceFullRepaint` — through the seam
`dispatch/{dispatch,input}.js` call `render()` directly (~30 sites);
`dispatch/effects.js` calls `forceFullRepaint`. Extend the seam with a
**synchronous** `paintNow()` (calls the registered render fn immediately — same
timing as the direct call, NOT the 50ms-debounced `scheduleRender`) plus a
`forceFullRepaint` slot. `dispatch` imports them from `leaves/render-queue`
(`dispatch→leaves`, legal). **No timing change** — `paintNow` is synchronous.

`app/suspend.js` + `app/tui.js` keep importing `render/paint` directly —
`app→render` is legal (app is the top that drives the compositor).

### The extended seam (`leaves/render-queue.js`)

```js
let _renderFn=null, _overlayFn=null, _forceFn=null, _invalidateFn=null;
function setRenderers({ render, overlay, forceFull, invalidate } = {}) { /* set slots */ }
function scheduleRender() { /* existing: 50ms debounce */ }
function scheduleOverlay() { /* existing: sync */ }
function paintNow()           { if (_renderFn) _renderFn(); }            // NEW — sync immediate
function forceFullRepaint()   { if (_forceFn) _forceFn(); }             // NEW
function invalidateRows(a,b)  { if (_invalidateFn) _invalidateFn(a,b); } // NEW
```

`paint.js` registers all four in its existing `setRenderers({...})` call.

## Stages (each independently green: suite + 9 smokes + bench parity)

1. **Move the pure leaves** — `scrollbar`, `painter`, `themes`, `render-queue`
   to `leaves/`; update importers. (No seam logic yet; `render-queue` still only
   `scheduleRender`/`scheduleOverlay`.)
2. **Extend the seam** — add `paintNow`/`forceFullRepaint`/`invalidateRows`
   slots to `leaves/render-queue`; `paint.setRenderers` registers them.
3. **Move `panel.js` → `leaves/draw.js`** with `setDimsProvider`; wire from
   `panel/api`; update the 12 overlay/panel/render importers.
4. **Split `decor`** — pure glyphs into `leaves/draw`, hit-tests into
   `panel/chrome-hittest`; repoint `input.js`.
5. **Reroute `dispatch`/`overlay`** off `render/paint`: `render()`→`paintNow()`,
   `forceFullRepaint`/`invalidateRows` via the seam; delete the static
   `render/paint` imports in `dispatch`/`overlay`.
6. **Verify** — `dep-walker` reports SCC `{dispatch, feature, overlay, panel}`
   (4), `render` out; suite/smoke/bench green. Update `PRINCIPLES.md §11/§12`,
   `blessed-exceptions.md`, this doc's status, and `docs/v0.6.5-tea-reaudit.md`
   F3 ledger row.

## Risks

- **Require cycles / TDZ.** The seam is required at module load by `paint.js`;
  keep `leaves/render-queue` zero-dep (it already is). The dims-provider closure
  in `panel/api` captures `getInstanceSlice` lazily — no eager `panel/route`
  load at draw-time.
- **Paint timing.** `paintNow` MUST be synchronous (not the debounced
  `scheduleRender`) to preserve the "handlers paint immediately" contract. The
  smokes (which assert post-input frame content) are the teeth.
- **`viewportDims` fallback order.** Provider-first, io/term-fallback — identical
  to today. Boot edge (before the layout slice seeds) still resolves via io/term.
- **Hot path.** `paintNow`/`invalidateRows` add one function-indirection hop;
  re-run `bench-*` to confirm parity (the route is the same as `scheduleRender`).

## Non-goals
- Dissolving `{dispatch, feature, overlay, panel}` (the domain tangle) — the
  separate, larger Cmd/port inversion. This arc deliberately stops at render-exit.
- Removing the `viewportDims` model-read — it stays in `panel` (correct home).
